API Reference / Publisher

Publisher Internal Only

Publishes posts to social platforms with retry logic and error handling.

⚠️ Internal Service

The Publisher is an internal service with no public REST API. It is triggered by Celery tasks and consumes from Kafka topics.

Architecture

ComponentDescription
Celery Queuepublish (priority 10 — highest)
Kafka ProducerPublishes to socialos.posts.published
Kafka ConsumerConsumes from socialos.posts.scheduled
Retry StrategyExponential backoff with jitter
Max Retries5 attempts

Platform Adapters

Each platform has a dedicated adapter implementing the BasePlatformPublisher interface.

PlatformAdapterFeatures
FacebookFacebookPublisherPages, Groups, Stories
InstagramInstagramPublisherFeed, Stories, Reels
Twitter/XTwitterPublisherTweets, Threads, Media
LinkedInLinkedInPublisherPosts, Articles, Company Pages

Base Publisher Interface

# apps/publisher/base.py

class BasePlatformPublisher:
    """Abstract base class for platform-specific publishers."""
    
    def publish(self, post: Post, account: SocialAccount) -> PublishResult:
        """Publish a post to the platform."""
        raise NotImplementedError
    
    def validate_content(self, post: Post) -> list[str]:
        """Validate post content against platform limits."""
        raise NotImplementedError
    
    def get_character_limit(self) -> int:
        """Return platform's character limit."""
        raise NotImplementedError

Retry Logic

AttemptDelayDescription
1ImmediateFirst attempt
230s + jitterFirst retry
32m + jitterSecond retry
48m + jitterThird retry
530m + jitterFinal retry

After 5 failed attempts, the post is marked as failed and an alert is sent to the organization admins.
