Scheduler API Coming Soon

Recurring post schedules using cron-based rules and timezone-aware scheduling.

🚧 Under Development

The Scheduler API is currently being implemented. This documentation outlines the planned endpoints and data models.

Architecture

ComponentDescription

Endpoints

MethodEndpointDescription
GET/api/v1/schedules/List all schedules
POST/api/v1/schedules/Create recurring schedule
GET/api/v1/schedules/{id}/Get schedule details
PATCH/api/v1/schedules/{id}/Update schedule
DELETE/api/v1/schedules/{id}/Delete schedule
POST/api/v1/schedules/{id}/pause/Pause schedule
POST/api/v1/schedules/{id}/resume/Resume schedule

Schedule Data Model

FieldTypeDescription
iduuidUnique schedule identifier
namestringSchedule name/label
cron_expressionstringCron expression (e.g., 0 9 * * 1-5)
timezonestringIANA timezone (e.g., America/New_York)
post_templateobjectTemplate for generated posts
platformsarrayTarget platforms
is_activebooleanWhether schedule is active
next_run_atdatetimeNext scheduled execution
last_run_atdatetimeLast execution time

Cron Expression Examples

ExpressionDescription
0 9 * * *Every day at 9:00 AM
0 9 * * 1-5Weekdays at 9:00 AM
0 9,17 * * *Every day at 9:00 AM and 5:00 PM
0 */2 * * *Every 2 hours
0 9 1 * *First day of every month at 9:00 AM
