Auth Core API v1

User registration, JWT authentication, password reset, email verification, and account management.

POST /api/v1/auth/signup/ Public

Register New User

Create a new user account. After successful registration, a verification email is sent to the provided email address. The user must verify their email before they can create or join organizations.

Request Body

FieldTypeDescription
email required string Valid email address. Must be unique across the platform.
password required string Minimum 10 characters. Must pass Django's password validation (common passwords rejected).
name required string Display name (max 255 characters).

Example Request

{
  "email": "john@example.com",
  "password": "SecurePass123!",
  "name": "John Doe"
}

Success Response (201 Created)

{
  "detail": "Account created. Please verify your email."
}

Error Responses

400 Bad Request 422 Validation Error 429 Rate Limited 
{
  "email": ["This email is already registered. Please sign in instead."]
}
POST /api/v1/auth/login/ Public

Login (Obtain JWT Tokens)

Authenticate with email and password to receive a JWT access/refresh token pair. The access token contains claims for org, role, name, email, and mfa_enabled.

Request Body

FieldTypeDescription
email required string Registered email address.
password required string Account password.

Example Request

{
  "email": "john@example.com",
  "password": "SecurePass123!"
}

Success Response (200 OK)

{
  "access": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "refresh": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
}

JWT Payload Claims: org (UUID|null), role (owner|admin|member|null), name, email, mfa_enabled (boolean)

Error Responses

401 Unauthorized 429 Rate Limited 
{
  "detail": "No active account found with the given credentials"
}
POST /api/v1/auth/refresh/ Public

Refresh Access Token

Exchange a valid refresh token for a new access token. The refresh token is rotated (blacklisted and replaced) on each use.

Request Body

FieldTypeDescription
refresh required string Valid refresh token from login or previous refresh.

Success Response (200 OK)

{
  "access": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "refresh": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
}

Error Responses

401 Token Invalid/Expired
POST /api/v1/auth/logout/ Auth Required

Logout (Blacklist Token)

Invalidate the current session by blacklisting the refresh token. The access token remains valid until expiry but cannot be refreshed.

Request Headers

HeaderValue
Authorization Bearer <access_token>

Request Body

FieldTypeDescription
refresh required string Refresh token to blacklist.

Success Response (204 No Content)

Empty response body.

Error Responses

400 Bad Request 401 Unauthorized
POST /api/v1/auth/forgot-password/ Public

Forgot Password

Initiate a password reset flow. If the email exists and the account is active, a reset link is sent. Previous unused reset tokens for this user are invalidated.

Request Body

FieldTypeDescription
email required string Email address of the account.

Success Response (200 OK)

{
  "detail": "If that email is registered, you will receive a reset link."
}

Error Responses

404 User Not Found 429 Rate Limited
POST /api/v1/auth/reset-password/ Public

Reset Password

Complete the password reset by providing the token from the email and a new password. Token expires after 1 hour and can only be used once.

Request Body

FieldTypeDescription
token required string Reset token from the email link.
password required string New password (min 10 characters, must pass validation).

Success Response (200 OK)

{
  "detail": "Password updated successfully."
}

Error Responses

400 Invalid/Expired Token 422 Validation Error
GET /api/v1/auth/verify-email/ Public

Verify Email

Verify the user's email address using the token from the verification email. Token expires after 24 hours and can only be used once.

Query Parameters

ParameterTypeDescription
token required string Verification token from the email link.

Example Request

GET /api/v1/auth/verify-email/?token=abc123xyz...

Success Response (200 OK)

{
  "detail": "Email verified successfully. You can now sign in."
}

Error Responses

400 Invalid/Expired/Used Token
POST /api/v1/auth/resend-verification/ Public

Resend Verification Email

Request a new verification email. Previous unused tokens are invalidated. Always returns 200 to prevent email enumeration attacks.

Request Body

FieldTypeDescription
email required string Email address to resend verification to.

Success Response (200 OK)

{
  "detail": "If that email is registered and unverified, a new link has been sent."
}

Rate Limiting

This endpoint has stricter rate limiting to prevent abuse. Limit: 3 requests per hour per IP.

DELETE /api/v1/auth/delete-account/ Auth Required

Delete Account

Permanently delete the authenticated user's account. This action is irreversible.

Behavior

  • If user is an owner of any organization, that organization is marked inactive and all members are notified.
  • If user is a member of organizations, the owners are notified of the departure.
  • All user data, tokens, and memberships are permanently deleted.
  • Posts authored by the user have their author_id set to NULL.

Request Headers

HeaderValue
Authorization Bearer <access_token>

Success Response (204 No Content)

Empty response body. Account has been deleted.

Error Responses

401 Unauthorized 404 User Not Found
